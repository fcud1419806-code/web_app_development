# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述
- **背景與動機**：學生族群通常收入固定但開銷來源多樣。建立一個輕量、簡單好用的記帳工具，能幫助他們培養良好的理財習慣，了解日常生活中的收支狀況。
- **目標用戶**：學生
- **核心價值主張**：提供一個直覺、簡單的工具，可以讓使用者進行日常記帳，輕易地瀏覽並編輯紀錄，隨時掌握財務狀況。

## 2. 功能需求

1. **日常記帳**
   - **使用者故事**：作為學生，我希望能夠快速輸入每天的收支項目與金額，以便精確記錄每筆日常花費。
2. **分類帳本**
   - **使用者故事**：作為學生，我希望能為每筆收支設定不同的分類（如：飲食、交通、娛樂、學習等），以便知道錢都花在哪裡。
3. **編輯與管理**
   - **使用者故事**：作為學生，我希望能夠隨時修改或刪除已經記錄的帳目，以便更正輸入錯誤或取消特定紀錄。
4. **預覽紀錄**
   - **使用者故事**：作為學生，我希望能隨時預覽歷史的收支明細清單，以便回顧過往的消費軌跡。
5. **計算與統計**
   - **使用者故事**：作為學生，我希望系統能自動計算某段時間內的總收入、總支出以及當前結餘，以便輕鬆掌握剩餘的生活費。

## 3. 非功能需求
- **技術限制**：使用前端網頁技術搭配後端 Flask + Jinja2 以及 SQLite 資料庫開發。
- **效能與安全考量**：
  - 確保使用者記帳資料寫入與讀取的正確性及穩定性。
  - 頁面加載速度應快速流暢，操作延遲極低。
  - 系統需具備基本的資料驗證機制，防止異常輸入（例如：金額不能為負數字元）。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 日常記帳功能（新增收支明細：項目名稱、金額、日期）。
  - 分類標籤選擇（內建基本的分類選項）。
  - 可以預覽所有的記帳清單。
  - 能夠編輯與刪除既有的單筆紀錄。
  - 自動計算總結餘與收支總額。
- **Should Have (應該有)**：
  - 圖表預覽（圓餅圖或長條圖）顯示各分類支出比例。
- **Nice to Have (有會更好)**：
  - 匯出為 CSV 功能。
  - 自訂新增個人分類。
